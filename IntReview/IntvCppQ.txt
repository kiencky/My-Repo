2. C++ Core Questions
■ 2. Sự khác nhau giữa stack và heap?

Trả lời:
Stack	Heap
Bộ nhớ cấp phát tự động	Cấp phát động bằng new/malloc
Nhanh	Chậm hơn
Tự giải phóng	Phải tự delete/free
Kích thước nhỏ	Lớn hơn

Ví dụ:
int a = 5;          // stack
int* p = new int;   // heap

■ 3. Constructor và Destructor dùng để làm gì?

Trả lời:
Constructor dùng để khởi tạo object.
Destructor dùng để giải phóng tài nguyên khi object bị hủy.

Ví dụ:
class A {
public:
    A() {
        cout << "Constructor";
    }

    ~A() {
        cout << "Destructor";
    }
};

■ 4. Virtual function là gì?

Trả lời:
Virtual function cho phép runtime polymorphism.

Ví dụ:
class Base {
public:
    virtual void show() {
        cout << "Base";
    }
};

class Derived : public Base {
public:
    void show() override {
        cout << "Derived";
    }
};

Nếu dùng:
Base* b = new Derived();
b->show();

→ output là "Derived".

■ 5. Pure virtual function là gì?

Trả lời:
Là virtual function không có implementation trong base class.
virtual void run() = 0;
Class chứa pure virtual function sẽ trở thành abstract class.

■ 6. Override và Overload khác nhau thế nào?

Override
Xảy ra giữa base và derived class.
Cùng signature.

Overload
Cùng tên hàm nhưng khác parameter.

Ví dụ:

void print(int);
void print(string);

■ 7. Deep copy và shallow copy?

Trả lời:
Shallow copy
Copy địa chỉ pointer.

Deep copy
Copy dữ liệu thật.

Ví dụ:

class A {
    int* data;
};

Nếu shallow copy:
2 object cùng trỏ 1 vùng nhớ
dễ double free

■ 8. Rule of 3 là gì?

Nếu class quản lý resource động thì nên implement:
Copy constructor
Copy assignment operator
Destructor

C++11 thêm:

Move constructor
Move assignment

→ thành Rule of 5.

■ 9. Smart pointer là gì?

Trả lời:
Smart pointer giúp quản lý memory tự động.

Các loại:
unique_ptr
shared_ptr
weak_ptr

Ví dụ:

unique_ptr<int> p = make_unique<int>(10);

■ 10. Sự khác nhau giữa unique_ptr và shared_ptr?
unique_ptr	shared_ptr
Chỉ 1 owner	Nhiều owner
Không copy được	Copy được
Nhẹ hơn	Có reference counting

■ 11. Reference và pointer khác nhau?
Reference	Pointer
Không null	Có thể null
Không đổi reference	Đổi địa chỉ được
Cú pháp đơn giản	Linh hoạt hơn

Bao gồm:

■ 15. vector và list khác nhau?
vector	list

random access nhanh	insert/delete giữa nhanh

■ 16. map và unordered_map khác nhau?

■ 17. RAII là gì?
Resource Acquisition Is Initialization.
Ý tưởng:

Ví dụ:

Đây là concept cực quan trọng trong modern C++.

36. Move semantics & rvalue reference là gì?
Cho phép "chuyển" tài nguyên thay vì copy. && là rvalue reference.
cppstd::string s2 = std::move(s1); // s1 rỗng sau dòng này
Dùng khi transfer ownership hoặc insert object lớn vào container.

37. Virtual destructor là gì? Tại sao quan trọng?
Nếu không có virtual ~Base() → delete qua base pointer chỉ gọi destructor của base → leak.
Quy tắc: class nào có virtual function thì destructor cũng phải virtual.

38. Undefined behavior (UB) là gì?
Hành vi không được định nghĩa bởi C++ standard — compiler có thể làm bất cứ điều gì.
Ví dụ: out-of-bounds, null dereference, use-after-free, data race, signed overflow.
Quan trọng trong automotive vì đây là lý do MISRA cấm nhiều feature của C++.

39. Template là gì?
Viết code generic, reuse với nhiều type. Instantiation tại compile time — không có overhead runtime.
cpptemplate<typename T>
T maxVal(T a, T b) { return a > b ? a : b; }
Lưu ý: thường đặt trong header file vì compiler cần thấy definition lúc instantiate.

Git
42. git cherry-pick dùng khi nào?
Lấy một commit cụ thể từ branch khác, không merge cả branch. Hay dùng để backport hotfix.

43. git bisect dùng để làm gì?
Tìm commit gây regression bằng binary search. Hữu ích trong large codebase.

44. Khi nào không nên dùng git rebase?
Không rebase trên shared branch vì sẽ viết lại history, người khác bị diverge. Chỉ rebase trên local branch của bản thân.

Automotive
45. Classic vs Adaptive AUTOSAR khác nhau thế nào?

Classic — bare-metal, RTOS, ECU truyền thống (ABS, engine control)
Adaptive — POSIX/Linux, service-oriented, dùng cho ADAS/autonomous driving


46. Hard vs Soft real-time là gì?

Hard — miss deadline = system failure (airbag, ABS). Dùng RTOS.
Soft — miss deadline = degraded performance (infotainment).

Linux thường không đủ cho hard real-time vì scheduler không đảm bảo latency.
